Housing



L’housing è un computer concesso in locazione ad uso esclusivo per l’allocazione del server del cliente. Il sito, dunque, non condivide le risorse del server con altri siti web. Una simile scelta comporta l'acquisto o l'affitto di uno o più server web nonché l'allestimento di un sistema adatto ad erogare il servizio.
Una volta acquistato il server ed i software e configurato l'ambiente è necessario sottoscrivere un contratto per il suo collegamento alla Rete ed eventualmente (nel caso dei servizi managed) per la sua manutenzione ed aggiornamento.

Scegliendo l'housing, in pratica, si hanno tutti i vantaggi di un ambiente dedicato con evidenti benefici in termini di prestazioni, personalizzazione e sicurezza, tuttavia ciò è controbilanciato dai costi elevati necessari per l'acquisto o l'affitto della macchina e dei software (sistema operativo, DBMS, web server, ecc.) nonché - nel caso dei servizi managed - per la loro installazione, manutenzione ed aggiornamento.


PRO

• Elevate performances

• Elevate possibilità di personalizzazione

• Maggiori livelli di sicurezza rispetto allo shared hosting

• Adatto a siti molto trafficati o che richiedono elevati livelli di prestazioni ed affidabilità


CONTRO

• Costi elevati per l'acquisto o l'affitto dell'hardware e dei software

• Richiede competenze sistemistiche o, in alternativa, ulteriori costi per la sottoscrizione di un servizio managed

• Struttura rigida e difficilmente scalabile

• Tempi di attivazione piuttosto lunghi

*Servizio Managed= è un servizio che viene preso in carico, erogato e controllato da un fornitore esterno (managed service provider).

*In molti casi l’housing viene confuso con il servizio di fornitura di un server dedicato. Pur utilizzando nei due casi, una macchina ad uso esclusivo, la differenza consiste proprio nel fatto che nell’housing il server è di proprietà del cliente mentre il server dedicato è di proprietà dell’hosting provider che lo concede in uso al cliente in una forma che richiama essenzialmente il noleggio.
